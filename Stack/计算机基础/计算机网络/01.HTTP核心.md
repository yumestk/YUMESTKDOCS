HTTP 1.0 和 2.0 有什么区别？
- HTTP的几个版本，其实就是围绕两个核心痛点演进的：连接成本（TCP链接多，慢）；头部冗余，队头阻塞，传输效率低
- HTTP1.0到1.1的核心升级，解决连接成本问题：最大的区别就是长连接（  Keep-Alive），HTTP1.0是短命的，每次请求都要走三次握手，四次挥手，效率低，HTTP1.1默认开启长连接，同一个TCP连接可以复用，不用频繁建连了；除此之外，在1.0时期一台服务器只能部署一个网站，1.1引入Host头，允许一台物理机部署多个虚拟主机（域名），这是现代云主机的基础
- HTTP1.1到2.0的核心革命，解决队头阻塞与带宽问题：
1. 多路复用：HTTP1.1虽然有长连接，但请求是串行，如果前面的请求卡住了，后面都得等，这叫队头阻塞，HTTP2.0引入了流（stream）的概念，他把一个TCP连接拆成了无数个逻辑上的流，每个请求打散成二进制帧乱序发送，到了服务端再组装，这就好比1.1是单车道，前面的车坏了后面全堵死；2.0是多车道高速公路，互不干扰
2. 二进制分帧：1.x是纯文本协议（肉眼可读），解析慢且容易出错，2.0全面改为二进制格式，机器解析更快更健壮
3. 头部压缩（HPACK）：1.1的Header往往很大（Cookie，User-Agent），而且每次请求都要重复发，浪费带宽，2.0在客户端和服务器端各维护一张索引表，重复的Header只需要发一个索引ID，大大减少了传输量
4. 服务器推送： 服务端可以预判你的需求，比如你请求了index.html，服务端知道你肯定还要style.css，就主动推给你，不用你再发请求来要（不过不好控制，Chrome等主流浏览器已经废弃了对它的支持）
