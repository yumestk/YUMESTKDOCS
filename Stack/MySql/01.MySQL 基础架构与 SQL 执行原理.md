**详细描述一条 SQL 语句在 MySQL 中的执行过程**
- MySQL架构分两层: **Server**层负责连接管理, SQL解析, 查询优化这些逻辑处理.**存储引擎层**负责数据的实际存取. 一条SQL从发出到拿到结果,分为这么几步:
1. 客户端先与MySQL建立连接,TCP握手,**连接器**负责验证账号密码,确认你能对哪些库表做什么操作
2. 老版本会先查缓存,看有没有一样的SQL缓存结果,8.0版本已经砍了,因为表一更新缓存就全失效,命中率太低了
3. **分析器**对SQL语句做词法,语法分析. 先把字符串拆成一个个token,识别出SELECT, 表名,列名,where条件元素;再按语法规则检查SQL写的对不对,最后生成一颗抽象语法树
4. **优化器**拿到语法树后,决定用哪个索引,最终形成一个执行计划
5. **执行器**按照计划,调用存储引擎的接口一行行地读数据,做条件过滤,最后把结果返回给客户端
- 连接器建连,分析器解析,优化器决策,执行器调度,存储引擎存取->这就是完整流程

MySQL 中的数据排序是怎么实现的
- 利用**索引天然排序**或者自己动手做`filesort`
- 如果order by的字段正好有索引,且排序方向一致,MySQL直接顺着 B+ 树叶子节点扫描就行, 不用额外排序, 效率很高
- 要是没命中索引,或者索引顺序不对,就得自己`filesort`,这时候MySQL会用`sort_buffer`这个内存缓冲区来排序,数据量小就在内存里搞定,数据量大塞不下就得借助磁盘做外部排序,用归并算法把小块排好序的数据合并成大文件
- 就像整理书架,有索引就直接按标签位置拿,顺序天然就是对的,没索引就自己一本本比大小,书少了内存里排就行,书多了就得搬到仓库分批整理再合并
- 内部排序分为单路和双路排序,单路:把所有的select放到`sort_buffer`排,排完直接返回. 双路排序只放排序字段和行id,排好序后再根据row_id去读其他字段
