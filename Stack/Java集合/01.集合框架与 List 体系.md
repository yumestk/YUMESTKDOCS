**Java 中有哪些集合类？请简单介绍**
- 主要分为Collection和Map,Collection用来存单个元素,Map用来存键值对
1. Collection分三条线:List, Set, Queue.     *List*有序可重复,ArrayList底层是数组,查询快增删慢. LinkedList底层是双向链表,增删快查询慢. *Set*不允许重复元素,HashSet基于哈希表,元素无序. LinkedHashSet用链表维护插入顺序. TreeSet基于红黑树,元素按大小排序,注意TreeSet是有序的.  *Queue*就是队列,PriorityQueue优先级队列,按元素大小出队;LinkedList也实现了Queue接口,可以当普通队列
2. Map分为HashMap, TreeMap, LinkedHashMap, ConcurrentHashMap. *HashMap*最常用,基于哈希表,键值对无序,key不能重复,允许null值null键. *TreeMap*基于红黑树,元素按大小排序.  *ConcurrentHashMap*是高并发场景的首选,分段锁设计. *LinkedHashMap*维护插入顺序,遍历时按放入顺序输出

**数组和链表在 Java 中的区别是什么？**
- 数组和链表最根本的区别在于**内存布局**, 数组是一整块连续的内存, 链表是散落在各处,靠指针串起来的节点. 数组算任意元素地址:基地址+下标*元素大小,随机访问是O(1)的

**Java 中的 List 接口有哪些实现类？**
- ArrayList, LinkedList, Vector, Stack, CopyOnWriteArrayList
- 1. ArrayList和LinkedList都不是线程安全的
- 2. 线程安全有两个选择: Vector是老古董,每个方法都加synchronized,性能很差; CopyOnWriteArrayList用写时复制,读不加锁,适合读多写少的场景

**Java 中 ArrayList 和 LinkedList 有什么区别？**
- 底层数据结构不同, 一个是动态数组,一个是双向链表,实际开发中,％90的情况都用ArrayList,因为现代CPU对连续内存的缓存命中率远高于链表这种跳来跳去的结构
- LinkedList每个节点包含前驱指针,后继指针和数据本身,节点散落在堆内存各处,靠指针穿起来
- LinkedList实现了Deque接口,栈和队列一般选Deque

**Java ArrayList 的扩容机制是什么？**
- ArrayList底层是个Object数组,当元素塞满了就得扩容,默认初始容量是10,扩容的新容量是原来的1.5倍,计算公式`oldCapacity + (oldCapacity >> 1)`
- 扩容流程:add时检查容量是否够用,如果不够就计算新容量,创建新数组并用Arrays.copyOf拷贝过去,elementData指向新数组
