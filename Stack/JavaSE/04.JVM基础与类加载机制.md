Java 的类加载过程是怎样的？
- 类加载就是把.class文件的二进制数据读进内存,经过校验,转换,最终变成JVM能用的Class对象
- 二进制流不一定非得来自.class文件,也可以是字节码工具动态生成的,或者从网络传过来的,只要格式对,JVM都认
- 整个类加载流程分为三个阶段:加载,连接,初始化.连接又能拆成验证,准备,解析三步
- 加载: 把二进制读进内存,在方法区生成类的运行时数据结构,同时在堆里创建一个Class对象作为访问入口
- 验证: 校验二进制流是否符合Class文件规范,包括魔数检查,版本号校验,元数据验证,字节码,符号引用验证,这一步是为了防止恶意代码搞崩JVM
- 准备: 给类变量(static修饰的变量)分配内存并设置初始零值,注意这里是零值,比如static int a = 123, 在准备阶段a的值是0,不是123,但如果是static final int a = 123,编译器就确定了,准备阶段直接123
- 解析: 把常量池里的符号引用换成直接引用,符号引用就是一个字符串形式的标识,比如java/lang/Object;直接引用是真正的内存地址或偏移量,能直接定位到目标1
- 初始化: 执行类构造器 `<clinit>()`方法,这时候才真正1执行static int a = 123这种赋值操作,静态代码块也是在这个阶段跑的

什么是 Java 中的双亲委派模型？
- 双亲委派模型是Java类加载的核心机制: 当一个类加载器收到加载请求时,不会自己先去加载,而是把请求向上委派给父加载器,一直委派到最顶层的Bootstrap ClassLoader,如果父加载器加载不了,才轮到自己加载
- 比如你写了个类叫MyService,Application ClassLoader收到加载请求后不会立刻加载,而是问Extension ClassLoader:你能加载吗; Extension Classloader又去问Bootstrap ClassLoader,BootStrap ClassLoader在rt.jar里找不到这个类,说我搞不定,请求就退回给Extension ClassLoader
- Extension ClassLoader在ext目录也找不到,再退回给Application ClassLoader,最终由Application ClassLoader从classpath里找到并加载

Java 中的序列化和反序列化是什么？
- 序列化就是把内存中的Java对象转成二进制字节流,这样才能存到硬盘或者通过网络发到别的机器,反序列化就是反过来,把字节流还原成Java对象,硬盘和网络只认0和1,不认Java对象,所以必须有这一层转换
- 怎么做序列化:类必须实现Serializable接口,这玩意就是许可证,没它Jdk序列化机制不让你序列化
- 怎么防止: 敏感字段比如密码,加个transient关键字,序列化时自动跳过
- 怎么稳: 显式定义serialVersionUID,相当于给类打个版本戳,防止改了类结构后反序列化旧数据时报错

你认为 Java 的优势是什么？
- 跨平台: Write once, Run anywhere;
- 垃圾回收机制(GC): Java会自动在后台帮你打扫内存垃圾,程序员只需专注于业务逻辑;
- 生态强大: 不管想做什么功能,基本都能找到现成的,成熟的开源库,就比如Spring全家桶,这就像搭积木,别人把积木块造好了,我拿来用就行,不用重复造轮子;
- 工程化友好: 强类型,面向对象封装,继承,多态这些特性,代码结构清晰,适合几百人协作的大型项目

JDK 和 JRE 有什么区别？
- JRE是Java Runtime Environment,是Java运行时环境,只能跑Java程序,JDK是Java开发工具包,能跑程序,写程序,编译程序,简单来讲,JDK包含了JRE再加上一堆开发工具
- JRE里面有什么: JVM,负责执行字节码; 核心类库,比如java.lang,java.util这些标准API; 一些配置文件和本地库
- JDK在JRE基础上多了: 编译器 javac, 把.java文件编成.class字节码; 调试器jdb, 排查问题用; 打包工具jar, 把一堆.class文件打成一个包; 各种监控诊断工具, 比如jps,jstack,jmap

你使用过哪些 JDK 提供的工具？
- 开发编译类:
1. javac,编译器,把.java源码编译成.class字节码
2. java,启动JVM跑字节码
3. jar, 打包工具, 把多个class文件压成一个jar包
4. javadoc, 从源码注释生成API文档
5. jdb, 命令行调试器, 可以设断点,单步执行
- 监控诊断类:
1. jps,列出当前机器上所有Java进程的pid和主类名
2. jinfo,查看和动态修改JVM参数
3. jstat,实时看GC情况,堆内存占用,类加载数量
4. jstack,导出线程栈,排查死锁,线程阻塞问题
5. jmap,导出堆快照或查看堆内存分布
- 性能分析类:
1. jconsole,图形化监控工具,能看内存,线程,CPU曲线
2. jvisualvm,功能更强的图形化工具,支持CPU采样,内存采样,线程分析
