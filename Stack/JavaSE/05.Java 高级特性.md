Java 泛型的作用是什么？
- 泛型就是把类型检查从运行时提前到编译时,编译器能在你写代码的时候就发现类型不匹配,不用等到程序跑起来就ClassCastException
- 类型安全: 编译期检查类型匹配,String集合塞不进Integer
- 消除强转: 取出元素时编译器自动插入类型转换代码,不用手动cast
- 代码复用: 一个泛型类可以处理各种类型, 不用为每种类型写一份代码

Java 泛型擦除是什么？
- 泛型擦除就是Java编译器在编译阶段把所有泛型信息都抹掉的过程,代码里写的`List<String>` `Map<Integer, User>`这些泛型,编译成class文件之后全变成了List, Map,泛型参数被替换成它的上界,没写上界默认Object
- 这就导致了几个问题: 运行时拿不到泛型的实际类型, 不能new T(), new T[], 不能对泛型使用instanceof,这些都是副作用1

什么是 Java 泛型的上下界限定符？
- 上下界限定符用来限制泛型参数的类型范围,让你在保证线程安全的同时获得更大的灵活性
- ? extends T 叫上界限定符, 表示类型必须是T或T的子类,主要用于读取场景
- ? super T 叫下界限定符, 表示类型必须是T或T的父类,主要用于写入场景

你使用过 Java 的反射机制吗？如何应用反射？
- 反射机制就是让程序在运行时能够 "照镜子" 看自己,动态获取类的结构信息,包括方法,字段,构造函数,还能直接操作它们.编译期不需要知道具体类型,运行时再决定要用哪个类,调哪个方法
- 反射核心流程:获取Class对象,通过类名字符串,.class语法或对象实例获取; 获取成员信息, 从Class对象中获取Field,Method,Constructor; 操作目标对象, 创建实例,读取字段, 调用方法

Java 中的注解原理是什么？
- 注解本质上是一个标记,是一种给代码添加元数据的机制,可以把注解打在类上,方法上,字段上,参数上,标记里还能带一些属性值
- 注解本身不会改变程序的运行逻辑,但编译器,框架,工具可以读取这些标记来做特定处理,比如编译时检查,生成代码,运行时做依赖注入

什么是 Java 中的动态代理？
- 动态代理的核心是运行时生成代理对象,不用提前写死代理类. 你想给某个对象的方法加点前置/后置逻辑, 比如打日志, 做权限校验, 开启事务, 但又不想改动原有代码,动态代理就是干这个的
- 和静态代理的区别就在于: 静态代理需要你手写一个代理类, 有10个接口就写是个代理类; 动态代理是JVM在运行时帮你生产代理类的字节码,一个InvocationHandler就能搞定所有接口
- 调用方发起请求,请求先到代理对象,代理对象执行前置逻辑(如打印日志,权限校验), 然后调用真实对象的方法,真实对象返回结果后,代理对象再执行后置逻辑(如记录耗时), 最后把结果返回给调用方

JDK 动态代理和 CGLIB 动态代理有什么区别？
- JDK动态代理基于接口,代理类必须实现至少一个接口,通过Proxy.newProxyInstance()生成代理对象,底层靠反射调用目标方法; 代理类实现目标接口,持有目标对象引用,方法调用时通过InvocationHandler转发,最终用反射调用目标方法
- CGLIB基于ASM字节码生成工具,通过继承目标类生成子类来实现代理,所以不需要接口,但final类和final方法无法代理; 代理类继承目标类,重写非final方法,方法调用时通过MethodInterceptor拦截,可以用InvokeSuper直接调用父类方法,不走反射

什么是 Java 的 SPI（Service Provider Interface）机制？
- SPI是JDK内置的一套服务发现机制,核心思想是接口定义在调用方,实现类放在服务提供方,运行时通过配置文件动态加载,调用方完全不用知道具体实现类是什么,只要实现方按规矩在META-INF/services目录下放个配置文件,ServiceLoader就能自动发现并加载
- JDBC就是SPI最经典的应用场景,JDK只定义了java.sql.Driver接口,MySQL,Oracle,PostgreSQL各自实现自己的驱动,你引入哪个jar包,DriverManager就自动加载哪个驱动,代码里完全不用写死驱动类名

Java 中的深拷贝和浅拷贝有什么区别？
- 浅拷贝就是复制对象本身,但引用类型的字段还是指向原来的内存地址,新对象和原对象共享内部的引用对象,改了一个另一个也跟着变
- 深拷贝就是把对象连同它内部所有引用对象都递归复制一遍,新旧对象完全独立,互不影响

Java 中 hashCode 和 equals 方法是什么？它们与 == 操作符有
什么区别？
- `==`比较的是内存地址,看两个引用类型是不是指向堆里同一块内存,对于基本类型,直接比值
- equals比较的是对象内容,Object类默认实现就是`==`,但通常会重写它来定义 "内容相等" 的逻辑,比如两个User对象,id一样就相等
- hashCode返回一个整数哈希值,主要给HashMap和HashSet这类哈希结构用,用来快速定位对象在数组中的槽位,哈希码不同,对象肯定不相等;哈希码相同,对象不一定相等

Java 中的 hashCode 和 equals 方法之间有什么关系？
- equals相等的对象,hashCode必须相等;hashCode相等的对象,equals不一定相等
- HashMap和HashSet判断两个对象是否相同,先比较hashCode,hashCode不等直接认为是两个不同对象,根本不会调equals,所以如果只重写equals不重写hashCode,集合就废了
