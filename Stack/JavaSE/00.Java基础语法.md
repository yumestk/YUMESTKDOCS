**Java 中的基本数据类型有哪些？**
- 8种基本数据类,按用途分为四类: 整型,浮点型,字符型,布尔型
1. 整型:byte 1字节存小数字(-128, 127), short 2字节基本没人用(-32768, 32767), int 4字节(+-21亿), long 8字节用来存大数字(+-922亿亿),赋值要加L后缀
2. 浮点型: float占4字节要加F后缀, double占8字节精度更高,是默认的小数类型
3. char:占两字节存一个Unicode字符,底层是一个无符号整数,范围(0, 65535)
4. boolean表示真假,JVM没定死大小,通常占1个字节


**Java 中的参数传递是按值还是按引用？**
- 参数传递只有按值传递, 不论是基本类型还是引用类型
1. 基本类型比如int这些,传递的是值的副本,也就是数值本身,对方法参数的任何修改都不会影响原始变量
2. 引用数据类型比如对象,数组,传递的是引用的副本,也就是对象引用的内存地址,方法内可以通过引用修改对象的属性,但不能改变引用本身,使其指向另一个对象


**Java 中的访问修饰符有哪些？**
- public, protected, default, private
 1. public: 没有访问限制, 任何位置都能访问
 2. protected: 同包下访问, 不同包只有子类能访问
 3. default: 只有同一个包下能访问, 出了包就看不见了
 4. private: 只能当前类用, 子类都不行

**Java 中静态方法和实例方法的区别是什么？**
- 静态方法属于类,不需要对象就能调用.实例方法属于对象,必须new对象才能调用
1. 静态方法用static修饰,通过类名直接调用, 在类加载时就存在, 不能用this,也不能访问直接访问实例变量和方法, 只能碰静态的
2. 实例方法不带static,必须通过对象调用, 能访问对象的实例变量,也能调用静态方法和变量, 静态和实例都能碰

**Java 中 for 循环与 foreach 循环的区别是什么？**
- for循环可以通过索引直接访问和修改元素,也可以倒着遍历,更加灵活
- foreach循环语法简洁:`for(String name : names)`,只能顺序遍历,拿不到当前索引,不能修改集合结构, 专门用来遍历数组或实现了iterable接口的集合

**Java Object 类中有什么方法，有什么作用？**
- Object类是Java所有类的根类,定义了11个方法,分为:对象通用操作,线程协作,对象生命周期
- 1. equals和hashcode是一对, 默认equals比的是内存地址,重写equals必须重写hashcode,不然放进HashMap会出问题,两个相等的对象可能放到不同的桶里
- 2. toString默认输出`类名@十六进制哈希码`,可读性差,重写后输出关键字段
- 3. wait,notify,notifyAll是线程间通信的基础原语,必须在synchronized块里调用,生产者消费者模型底层就是这几个方法实现的
- 4. getClass返回运行时类型信息, 是反射的入库, Spring框架大量用它来做依赖注入和AOP
- 5. clone默认是浅拷贝,对象里的引用类型字段只拷贝地址.想深拷贝得自己实现逻辑,或者用序列化
- 6. finalize已经标记废弃了,资源清理的执行时机不可控,GC半天都调不通.用try-with-resources或者实现AutoCloseable才是正道

**Java 中的字节码是什么？**
- 字节码是java源代码编译后产生的**中间代码**,存储在.class文件里.它不是跑在CPU的机器码,而是给JVM看的指令集
- Java的一次编写,到处运行就是字节码实现的.源代码用javac编译成字节码,字节码与平台无关;然后不同平台的JVM负责把字节码翻译成对应平台的机器码执行
- 字节码是基于栈的指令集,绝大数操作都是围绕操作数栈执行;从局部变量表加载数据到栈上,在栈上做运算,再把结果存回去或者返回

**什么是 Java 中的迭代器（Iterator）？**
- Iterator就是Java集合框架提供的统一遍历接口,不管底层是哪个集合,都能用同一套API依次访问每个元素,完全不用关心是数组还是链表
- 核心方法就三个:hasNext()判断是否有下一个元素,next()取出下一个元素,remove()删除刚刚取出来的元素
- ListIterator可以双向遍历,有next()和previous方法


**栈和队列在 Java 中的区别是什么？**
- Stack和Queue查看栈顶和查看队头都是用peek,就拿起来看一眼,不做修改
- Stack的push是入栈,pop出栈. Queue的offer是入队,poll出队1
- 使用场景上,栈适合需要"回退"的场景,比如浏览器前进后退,方法调用栈,DFS深度优先搜索;队列适合需要 "排队" 的场景,比如线程池任务队列,消息中间件,BFS广度搜索

**什么是 Java 中的自动装箱和拆箱？**
- 自动装箱就是Java编译器把基本类型转为对应的包装类.自动拆箱就是反过来
- 基本类型int通过Integer.valueOf()转换为Integer对象.
- Integer对象通过intValue()转换回int基本类型
- 集合类比如ArrayList只能存对象,所以add(10)自动做装箱
- 从集合取出做算术运算会自动拆箱

**为什么在 Java 中编写代码时会遇到乱码问题？**
- 编码和解码用的字符集不一致

**Float 经过一系列操作后，如何判断是否和另一个数相等？**
- 浮点数进行运算后,绝对不能用 == 比较,因为二进制无法精确表示所有十进制小数
- 三种解决:阔值比较法,设一个误差容忍范围;BigDecimal:需要精确计算时,用BigDecimal存储和运算;整数化处理

**如何在 Java 中调用外部可执行程序或系统命令？**
- Runtime.exec()和ProcessBuilder.ProcessBuilder功能更强,推荐使用
- `Process process = Runtime.getRuntime().exec("filename.txt")`
- ProcessBuilder比Runtime更加灵活,可以设置工作目的,环境变量,重定向输入输出

**Java 的 Optional 类是什么？它有什么用？**
- Optional类是Java8引入的容器类,专门用来包装可能为空的值.核心作用不是消灭空指针,而是清晰的表达这个值可能不存在的语义
- 核心API:Optional.of(value)包装非空值,传null直接npe;Optional.ofNullable(value)包装可能为空的值,传null返回空Optional;Optional.empty()创建一个空Optional
- orElse():不管Optional有没有值,都会执行后面的操作
- orElseGet():有值才会执行后面的操作

> - foreach 基于迭代器实现，集合的 remove 方法会修改 modCount，导致迭代器检测到版本不一致，抛出并发修改异常。正确做法是使用迭代器的 remove 或 removeIf。
