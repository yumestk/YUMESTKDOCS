**什么是 Java 的封装特性？**
- 封装就是把数据和操作数据封装在一起,对外只暴露必要的接口,内部实现细节不让外部直接碰.Java通过访问修饰符实现封装,字段用private藏起来,通过public的getter/setter方法控制访问
- 封装的核心价值在于控制,你控制了数据怎么被读取,怎么被修改,就控制住了数据的正确性

**什么是 Java 中的继承机制？**
- 允许子类复用父类的字段和方法,用extends关键字建立继承关系.Java只支持单继承,一个类最多一个直接父类,但可以通过继承链形成多层结构

**什么是 Java 的多态特性？**
- 父类引用指向子类对象,调用同一个方法时,会执行子类自己的实现
- 一条指令,多种表现,编译看左边,运行看右边,底层是通过虚方法表vtable动态绑定找到真正要执行的方法
- 多态的三个条件:要有继承或实现关系,extends or implement; 子类要重写override父类方法;父类引用指向子类对象,也叫向上转型

Java 方法重载和方法重写之间的区别是什么？
- 方法重载在一个类里,允许多个同名方法,只要参数列表不同
- 方法重写发生在继承关系中,子类重写父类的某个方法,参数列表和方法名完全相同

接口和抽象类有什么区别？
- 接口的设计是自上而下的,我们知晓某一行为,于是基于这些行为约束定义了接口,一些类需要有这些行为,因此实现对应的接口
- 抽象类是自下而上的,我们写了很多类,发现它们之间有共性,很多代码可复用,因此将公共逻辑封装成一个抽象类
- 自上而下就是先约定接口,再实现; 自下而上就是先有一些类,才抽象了共同父类
- 接口中的方法默认是public abstract,Java8之后可以设置default方法或静态方法,抽象类可以包含abstract方法和具体方法;接口中的成员变量默认是public static final,也就是常量,接口不能包含构造方法;抽象类可以包含构造函数

什么是 Java 内部类？它有什么作用？
- 把逻辑相关的类放一起,提高内聚性,减少冗余代码,匿名内部类在事件监听,回调这些场景用的很多
- 成员内部类: 定义在类里面不带static修饰,可以访问外部类的所有成员
- 静态内部类: 用static修饰,只能访问外部类的静态成员
- 局部内部类: 定义在方法里面,只在这个方法可见,实际用的较少
- 匿名内部类: 没有类名,在创建对象时定义,用的最多的场景是实现接口或继承抽象类,比如回调,监听器

为什么 Java 不支持多重继承？
- 因为会导致菱形继承问题,如果D,同时继承B和C,调用B和C方法名相同的方法,此时会产生歧义,到底该执行哪个

Java 面向对象编程与面向过程编程的区别是什么？
- 面向对象编程把类或对象作为基本单元来组织代码,围绕封装,继承和多态来设计,面向过程编程以过程或函数为中心,过程就是动作,对应到代码里就是函数,函数和数据是分离的
