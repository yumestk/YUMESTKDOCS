**Java 中 String、StringBuffer 和 StringBuilder 的区别是什么？**
- 三者都用来处理字符串,核心区别在于可变性和线程安全
- String是不可变的,底层的char数组用final修饰,一旦创建就定死在那,每次拼接,替换都会创建一个新对象,原来那个不会变
- StringBuffer和StringBuilder都是可变的,底层用一个可扩容的数组存字符,区别在于StringBuffer的方法加了synchronized,是线程安全的;StringBuilder没加锁,单线程下性能更好
- 字符串基本不变用String;多线程下要频繁修改字符串就StringBuffer,单线程就StringBuilder

**Java 的 StringBuilder 是怎么实现的？**
- StringBuilder底层就是一个可扩展的字符数组加一个count变量
- append操作流程: 先算一下要追加的内容转成字符需要几位,检查当前数组容量够不够,不够就扩容,把内容拷贝到数组末尾,更新count值
- 扩容策略是原容量 * 2再 + 2,不传参默认大小16个字符

**为什么 JDK 9 中将 String 的 char 数组改为 byte 数组？**
- 为了省内存,char[]用UTF-16编码,每个字符固定2字节,但实际大量应用场景里,字符串都是纯英文或数字,比如URL,Json的Key,日志内容,这些用1个字节就够

**Java 中包装类型和基本类型的区别是什么？**
- 基本类型直接存数值,包装类型是对象,这就是最本质的区别
- 性能差距很明显,int就占4字节,Integer对象在64位JVM上光对象头就占16字节(开启指针压缩12字节),加上4字节的int值,还要4字节来**对齐**,一共24字节
- 包装类型比较一律用equals

**什么是 Java 的 Integer 缓存池？**
- Integer 缓存池是JVM为了减少小整数对象的重复创建而设计的优化机制.-128到127范围内的Integer对象会被先创建并缓存起来,自动装箱时直接返回缓存对象,不用每次都new个新的

**什么是 Java 的 BigDecimal？**
- BigDecimal 是 Java 提供的高精度数值计算类,专门解决float和double的精度丢失问题

**BigDecimal 为什么能保证精度不丢失？**
- 因为压根不用浮点数那套二进制表示法,而是用整数加小数点位置的方式来存储数值
- 内部就两个关键字段: intVal是一个BigInteger,存的是把小数点去掉后的整数;scale记录小数点往左移了几位

**使用 new String("yupi") 语句在 Java 中会创建多少个对象？**
- 一个或两个,取决于字符串常量池里有没有"yupi"这个对象的引用
- 如果常量池里没有,那JVM会先在堆上创建一个字符串对象放进常量池,然后new String()又在堆上创建一个新对象,一共2个
- 如果常量池中已经有了,new String()只会在堆上再创建一个对象,常量池那边直接复用
- 核心就是ldc指令的行为,它会检查常量池,没有就创建并缓存,有就直接返回引用,所以第一次遇到字面量 "yupi" 时会触发一次堆上对象的创建,而new String()必定再创建一个

**什么是 Java 中的不可变类？**
- 不可变类就是一旦创建完成,状态就定死在那,后面没法再改,最典型就是String,还有Integer这种基本类型的包装类
- 五个条件:类声明为final,防止子类继承后搞出可变行为;所有字段都是private final,只能在构造阶段赋值一次;通过构造函数初始化所有字段;不提供任何setter方法或者修改状态的方法;如果字段引用了可变对象,getter必须返回副本,不能把原对象暴露出去
- 天然线程安全: 状态不能改,多线程随便访问,根本不用加锁
- 适合做缓存Key: 比如HashMap的Key要求hashCode稳定,不可变对象天然满足;代价就是每次"修改"都得创建新对象,String拼接频繁的场景内存压力不小,所以才有StringBuilder
