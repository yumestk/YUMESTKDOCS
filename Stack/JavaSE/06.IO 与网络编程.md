Java 的 I/O 流是什么？
- Java的I/O流就是一套用来读写数据的类库,程序通过他从文件,网络等地方读数据进来,或者把数据写出去
- IO流按数据单位分为两大类: 字节流和字符流
- 字节流处理的是8位的原始字节,适合处理图片,视频,压缩包这类二进制文件,核心类是InputStream和OutputStream; 字符流处理的是16位的Unicode字符,专门用来读写文本文件,核心类是Reader和Writer

什么是 Java 的网络编程？
- Java网络编程就是用java.net包提供的类来实现网络通信,最核心的就是Socket编程,客户端用Socket连接服务器,服务器用ServerSocket监听端口接收连接,双方通过输入输出流收发数据
- TCP通信基本流程:服务端先创建ServerSocket绑定端口开始监听,客户端创建Socket指定服务端IP的端口发起连接,服务端accept方法返回一个新的Socket代表这个连接,之后双方通过各自Socket的输入输出流进行通信,用完后关闭连接
- 底层原理上,Socket是对操作系统TCP/IP协议栈的封装,创建Socket时JVM调用操作系统的socket系统调用,connect时触发TCP三次握手,close时触发四次挥手,数据传输走操作系统内核的TCP缓冲区,write把数据写到发送缓冲区就返回,内核负责把数据切分成报文发出去

什么是 BIO、NIO、AIO？
- BIO,NIO,AIO是Java里三种不同的IO模型,核心区别在于线程在等待数据时的行为
- BIO(Blocking)是同步阻塞模型,线程发起read就卡在那,数据没来之前啥也干不了,一个连接配一个线程,1000个连接就得开1000个线程,线程切换开销直接把CPU拖垮
- NIO(Non-Blocking)是同步非阻塞模型,线程可以先去干别的,通过Selector轮询哪些Channel有数据可读,一个线程能管几千个连接,Netty,Tomcat的NIO模式都是这个套路
- AIO(Asynchronous)是异步非阻塞模型,发起读请求后直接返回,操作系统把数据拷贝完了再通过回调通知你,Linux下AIO支持一般,实际用的不多
- BIO就是你去银行排队,轮到你才能办业务; NIO是你取个号,叫号了再去柜台了AIO是你把单子给大堂经理,办完了她打电话通知你

什么是 Channel？
- 是Java NIO里的核心组件,可以理解为数据传输的双向通道,和传统的Input/OutputStream不一样,Channel天生就是双向的,一个通道既能读也能写,不用分别搞输入输出流
- 传统IO是单向的,InputStream只能读,OutputStream只能写,要实现双向通信,必须同时持有两个对象
- Channel是双向的,一个Channel对象既可以read也可以write,网络通信只需要一个SocketChannel就能搞定收发
- Channel本身不存数据,它必须配合Buffer使用,读数据是从Channel读到Buffer里,写数据就是把Buffer的内容写到Channel里
- 常用的Channel: FileChannel-专门用来读写文件,支持内存映射和零拷贝,Kafka和RocketMQ底层就靠它来实现高性能磁盘读写
- SocketChannel: TCP客户端通道,连上服务器之后用它收发数据
- ServerSocketChannel: TCP服务端通道,负责监听端口,接收新连接,每接收一个连接就生成一个SocketChannel
- DatagramChannel: UDP通道,不需要建立连接,直接发数据报

什么是 Selector？
- Selector是Java NIO里实现IO多路复用的核心组件,一个线程通过Selector就能同时监听成百上千个Channel的读写事件,不用给每个连接开一个线程
- Selector就是为了解决传统BIO模型来一个连接就得开一个线程守着,1000个连接就要1000个线程,线程切换的开销比干正事的开销还大; Tomcat NIO模式,Netty,Redis的网络层都是基于这套机制
- Selector模型:一个线程持有一个Selector,多个Channel注册到Selector上,Selector轮询检测哪些Channel有事件,有事件才去处理,没事件就等着
- Selector能监听4种事件: OP_ACCEPT-ServerSocketChannel专用,表示有新连接进来了; OP_CONNECT-客户端专用,表示连接建立完成; OP_READ-通道里有数据可读; OP_WRITE-通道可以写数据了,发送缓冲区有空间
