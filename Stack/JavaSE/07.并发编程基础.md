如果一个线程在 Java 中被两次调用 start() 方法，会发生什么？
- 直接抛 lllegelThreadStateException,Java规定一个Thread对象只能调用一次start(),第二次调用时线程状态已经不是 NEW 了,JVM会拒绝启动并抛异常
- 线程调用start()后状态从 NEW 变成 RUNNABLE.执行完run()方法后变成TERMINATED,不管是RUNNABLE还是TERMINATED,再调start()都会因为threadStatus != 0 而抛异常
- 线程状态是单向流转的,一旦离开NEW状态就回不去了.这和操作系统线程设计有关: 一个线程执行完毕后,它的栈空间,寄存器上下文都被回收了,没法复活

Java 中 wait() 和 sleep() 的区别？
- 就一个区别,wait()会释放锁,sleep()不会释放锁
- wait()是Object类的方法,必须在synchronized块里调用调用后当前对象释放对象锁,进入等待队列,等着别忍调notify()或notifyAll()把它唤醒
- sleep()是Thread类的静态方法,随便在哪都能调,线程睡指定时间自己醒,如果当前线程有锁,睡觉期间抱着锁睡觉,别的线程进不来
