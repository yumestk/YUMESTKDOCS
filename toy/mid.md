# 中小厂日常实习面试押题（精简版）

> 候选人：彭锦宣（小Y）｜ 目标岗位：AI Agent / Java 后端开发
> 难度定位：大二日常实习，中小厂（非字节/阿里/腾讯级别）
> 核心原则：能讲清楚自己做了什么 > 背八股。中小厂更看重"能不能干活"而不是"理论有多深"。

---

## 一、自我介绍阶段押题

中小厂面试官通常不会在自我介绍阶段就高强度追问。他们更关注：你做了什么、你用的技术栈和岗位是否匹配、你表达是否清晰。

---

### 追问点 1：两个项目，挑一个你最满意的详细讲讲

**主问题：** 你说有两个项目——如果只让你挑一个最能代表你水平的，你选哪个？为什么？

**考察意图：** 中小厂面试官不指望两个都了解透，他们想知道你对自己做的东西有没有判断力。能说清楚"为什么这个比那个好"比两个都泛泛而谈强。

**追问（1 层）：** 你刚才说最满意的是 XX 项目——那这个项目里你写的最满意的一段代码是什么？不用背代码，讲思路就行。

---

### 追问点 2：你提到了"参考 OpenManus"——你是看文档学的还是看源码学的？

**主问题：** 你参考 OpenManus 做自己的项目——那你具体参考了它的哪些部分？有没有哪段源码你印象特别深？

**考察意图：** 区分"跑了个 demo 改改"和"真的研究过别人怎么写的"。中小厂面试官希望你是一个"能自己看代码学习"的人，因为他们没有大厂那么完善的培训体系。

---

### 追问点 3：这些技术栈（Spring AI / LangChain4j / PGvector）你是怎么学的？

**主问题：** 你才大二，简历上这么多技术——你能挑一个，说说从零到能用在项目里花了多久、踩过什么坑吗？

**考察意图：** 中小厂非常看重自学能力。面试官想确认你不是培训班流水线出来的，而是真的有能力自己啃新技术。

---

## 二、项目深挖（精简版，每个项目 4 问）

---

### 项目一：YumeManus（AI 智能体）

#### 第一问（破题，让面试官理解你做了什么）

> "用一句话告诉我，你这个 Agent 和直接调 ChatGPT API 有什么区别？"

**考察意图：** 如果你能说清楚"我的 Agent 多了记忆、能自己调用工具、能多步推理"，说明你理解 Agent 的核心价值。如果只能说"就是接了个 API"，那就完了。

---

#### 第二问（核心技术点）

> "你的 Agent 调用工具（Tool Calling）的时候，整个流程是怎么走的？比如用户说'帮我查一下北京天气然后推荐穿什么'，从用户发消息到最终回复，Agent 内部发生了什么？"

**考察意图：** 这是 Agent 项目最核心的一问。中小厂面试官不要求你画状态机，但要求你能把流程讲通。

**预期回答要点：**
1. 用户消息 + 历史对话 → 拼成 messages
2. 发给 LLM，LLM 返回"我需要调用天气工具"
3. Agent 解析返回结果，实际调用天气 API
4. 把 API 结果拼回 messages，再发给 LLM
5. LLM 综合天气信息，给出穿衣建议

能讲清楚这个闭环就及格。如果能补充"如果 API 挂了怎么办"直接加分。

---

#### 第三问（数据与存储）

> "用户的对话历史你是怎么存的？重启服务后还能恢复吗？"

**考察意图：** 中小厂面试官很务实——他们想知道你的系统是不是"重启就没"的玩具。

**追问（如果上面答得好）：** 存对话历史的时候，你是存原始 JSON 还是序列化对象？为什么这么选？

---

#### 第四问（自评与复盘）

> "如果让你现在重新做这个项目，你会改什么地方？"

**考察意图：** 能诚实说出自己做得不好的地方，说明你真的思考过。中小厂面试官反而怕那种"我项目完美无缺"的候选人。

---

### 项目二：Yume No Code（AI 零代码平台）

#### 第一问（破题）

> "用户输入'帮我做一个个人博客首页'——你的系统从收到这个请求到返回一个能访问的网页，中间经历了哪些步骤？"

**考察意图：** 和项目一一样，考察你能不能把整个流程讲通。中小厂面试官可能不懂 LangGraph4j，但他们听得懂"先做 A，再做 B，最后 C"。

---

#### 第二问（核心技术点）

> "AI 生成的代码有时候会有 bug 或者跑不起来——你是怎么处理这种情况的？"

**考察意图：** 这暴露了一个真实的工程问题。如果你的回答是"我写了个质量检查的步骤，如果代码有语法错误就重新生成"，说明你确实考虑过可靠性。如果是"基本都能跑"，面试官会觉得你没怎么测试过。

---

#### 第三问（性能相关）

> "你的项目里提到用了缓存和索引优化——能举一个具体的例子吗？比如哪个查询慢了，你做了什么让它变快了？"

**考察意图：** 中小厂面试官喜欢听到"有数据、有对比"的回答。如果你能说出"查对话历史原来要 800ms，加了索引后变成 200ms"，比说"我优化了性能"强 10 倍。

---

#### 第四问（自评）

> "你觉得这个项目最不成熟的地方是什么？如果有用户真的想用，哪里会出问题？"

**考察意图：** 同上——诚实自评比吹牛重要。中小厂面试官自己也是写代码的，听得出来你在说真话还是假话。

---

### 穿插八股追问（项目关联，减轻深度）

面试官可能会顺着你的项目问一些基础八股，但不会很深。以下是最可能被问到的：

| 八股问题 | 难度 | 怎么准备 |
|---------|------|---------|
| "你用了 MySQL，能说说索引是什么吗？为什么能加速查询？" | 基础 | 一句话：索引就像书的目录，B+ 树结构让查找从 O(n) 变成 O(log n) |
| "你用了 Redis，Redis 和 MySQL 的区别是什么？什么时候用 Redis？" | 基础 | Redis 快但是贵（内存）、数据可能丢；MySQL 慢但是持久。缓存热点数据用 Redis |
| "Spring Boot 的 `@Autowired` 是干嘛的？" | 基础 | 自动注入依赖，不用自己 new 对象 |
| "Git 你常用哪些命令？" | 基础 | clone/add/commit/push/pull/branch/merge，能说出 5 个就行 |
| "你项目里提到设计模式——能举一个你真实用过的例子吗？" | 基础 | 用你自己的代码讲，不要背定义 |
| "你用了 RocketMQ——消息队列是干什么的？和直接异步执行有什么区别？" | 基础 | MQ 持久化消息、解耦生产者和消费者、服务重启不丢任务 |

## 三、算法手撕押题（中小厂难度）

中小厂实习面试的算法题通常很简单，可能就一道，甚至没有。即使有，也基本是 LeetCode Easy 到 Medium 偏易的水平。

---

### 题一：反转字符串中的单词顺序

**题目描述：** 给定一个字符串，单词之间用空格分隔，反转单词的顺序。比如 `"I love coding"` → `"coding love I"`。

**面试官想看什么：** 基本的字符串操作、边界处理（首尾空格、连续空格）。

**实现要点：** `split(" ")` → 反转数组 → `join(" ")`。如果用 Java 手写，用 `StringBuilder` 从后往前遍历。O(n) 时间，O(n) 空间。

---

### 题二：判断有效括号

**题目描述：** 给定一个只包含 `()`、`{}`、`[]` 的字符串，判断括号是否有效配对。比如 `"()[]{}"` → true，`"([)]"` → false。

**面试官想看什么：** 栈的运用。这是最经典的栈题目，面试官也知道你大概率见过——就看你能不能写对、注意边界。

**实现要点：** 遍历字符，左括号入栈，右括号检查栈顶是否匹配。最后栈为空则有效。注意空字符串返回 true。

---

### 题三（轻量设计）：带过期时间的简单缓存

**题目描述：** 设计一个最简单的缓存类，能存键值对，每个键有过期时间，过期了就自动失效。不需要考虑 LRU、不需要线程安全。

**面试官想看什么：** 不是考算法，是考"你能不能把一个需求变成能跑的代码"。方法命名、边界处理比性能重要。

**实现要点：**
```
class SimpleCache<K, V> {
    Map<K, V> data = new HashMap<>();
    Map<K, Long> expireAt = new HashMap<>();
    
    void put(K key, V value, long ttlMs) { ... }
    V get(K key) {
        // 检查是否过期，过期返回 null 并清理
    }
}
```
不需要 LinkedHashMap，不需要任何高级数据结构。写出能跑的、逻辑清楚的代码就行。

## 四、日常问题押题

中小厂的日常问题和大厂不一样——大厂问"你为什么选我们"，中小厂问"你能待多久"。

---

### 1. "你才大二，为什么这么早出来实习？"

**隐藏考察点：** 中小厂招实习生成本高（需要人带），他们怕你干两个月就跑。面试官想确认你是不是认真的。

**回答策略：** 诚实说你的动机。比如"我觉得学校里学的东西和工业界差太远，想早点接触真实项目"。如果能说"我暑假/寒假可以全职，开学后也能保证每周 3-4 天"，直接打消对方顾虑。

---

### 2. "你期望从这段实习里学到什么？"

**隐藏考察点：** 中小厂没有大厂的培训体系，他们希望实习生"带着明确目标来"而不是"等着被安排"。如果回答太模糊（"想学东西"），面试官会觉得你不知道自己要什么。

**回答策略：** 具体到 1-2 个点。比如"我想看看真实的 AI Agent 项目在生产环境是怎么部署和运维的，因为我目前只在本地跑过"或者"我想学习团队协作的流程，Git 分支管理、Code Review、需求评审这些我都没经历过"。

---

### 3. "如果你遇到一个不会的技术问题，你一般怎么解决？"

**隐藏考察点：** 中小厂带你的 mentor 自己也很忙，不可能手把手教你。面试官想看你的独立解决问题能力。

**回答策略：** 给一个具体流程而不是泛泛说"搜 Google"。比如："先看官方文档 → 搜 GitHub Issues → 看源码相关部分 → 如果实在不行再问 mentor，但我会带着'我已经试过 A、B、C 都不行'的上下文去问"。

---

### 4. "你一个人做项目的时候，有没有做到一半想放弃的时候？怎么坚持下来的？"

**隐藏考察点：** 中小厂的实习项目可能比较枯燥（增删改查居多），面试官想看你的抗压能力和坚持度。

**回答策略：** 讲一个真实的、但不致命的困难。比如"我刚开始做 YumeManus 的时候，Agent Loop 一直在死循环，调了三天都没解决，后来我发现是 prompt 里少了一句'如果已经完成就输出 FINISH'。"——重点不是困难多大，而是你怎么解决的。

---

### 5. "除了写代码，你还有什么兴趣爱好？"

**隐藏考察点：** 这是一个"软着陆"问题，出现在面试最后几分钟。面试官想看你这个人好不好相处——中小厂团队小，一个不合群的人影响很大。

**回答策略：** 轻松答，不要硬凹"我业余时间都在看技术博客"。打球、看动漫、打游戏都可以说——真诚比政治正确重要。

---

## 五、高危预警清单（中小厂版）

中小厂面试官和大厂不一样：大厂面试官见多了各种简历，遇到吹牛的会往死里压；中小厂面试官更看重"诚实"，一旦发现你在某个点撒谎，直接挂。

---

### 🔴 高危 1：技术名词堆砌（最致命）

**风险：** 简历上写了"掌握/了解"十几个技术名词，但真被问到其中任何一个，只能说一句"我就用过一下"。

**紧急补救：** 把简历上每个技术名词过一遍——如果某个技术你说不出"我在 XX 项目的 XX 场景用到了它，解决了 XX 问题"，那就从简历上删掉，或者改成"了解"后面加一句"在项目中未使用"。中小厂面试官宁可你技术栈窄但扎实，也不要宽但全是空壳。

---

### 🔴 高危 2：项目细节说不清楚

**风险：** 如果面试官问你"这个项目你自己写了多少行代码？哪部分是抄的？"——你答不上来，说明你没真做。

**紧急补救：** 面试前打开你自己的项目，选一个模块从头到尾看一遍代码。不需要记住每一行，但至少要能说出"这个类干什么、那个方法负责什么逻辑"。如果你自己看代码都懵——那面试一定会翻车。

---

### 🔴 高危 3："开源"但没有 README

**风险：** 中小厂面试官大概率不会提前看你的 GitHub，但如果你提到"开源"，他可能会当场打开看一眼。如果 README 是空的或者只有一句话，印象分直接扣光。

**紧急补救：** 面试前给两个项目的 README 各加一段话：项目简介、技术栈、怎么运行、截图（哪怕只有一张）。15 分钟就能搞定的事，千万不要省。

---

### 🔴 高危 4："信息安全"专业但项目全是 AI — 真实情况版

**你的真实情况：** 后端是早早就自学的，安全只是高考录的专业。对 AI 感兴趣是因为接触了 openclaw 开源项目。

**面试回答模板：** "我高考录的是信息安全，但实际上从大一就开始自学 Java 后端了。后来接触到 openclaw 开源项目，发现 AI Agent 能把后端和 AI 结合起来，产生了很强兴趣。"

**openclaw 避坑（自我介绍只说一次，立刻转场）：** 万一面试官追问"那你了解 openclaw 源码吗"，诚实说"没有深入读过源码，主要看的是文档，我的项目参考的是 OpenManus 架构从零写的"。不要硬装读过源码。

---

### 🔴 高危 5（新增）：RocketMQ —— 能说清楚"为什么用"和"怎么用"吗？

**风险：** 新加了 RocketMQ，但可能只是配了个依赖、写了 Producer/Consumer 就完事了。中小厂面试官如果自己也用过 MQ，会追问基本原理。

**紧急补救：**
- 能一句话说清楚：为什么用 RocketMQ？——"之前部署一个网站要等 LangGraph4j 全流程跑完，用户干等 1-2 分钟。改成 MQ 异步后，用户提交请求立刻返回 '任务已提交'，后台消费消息执行部署。"
- 知道 Topic / Producer / Consumer 是什么——不需要深，但要能说
- 如果面试官问"MQ 挂了怎么办"，诚实回答"我目前只做了基础实现，生产级还需要加死信队列和监控告警"——比硬编强

---

### 🟡 附：中小厂面试特别提醒

1. **态度比技术重要**：大厂面试官会说"你这个答案不够深入"，中小厂面试官更看重"你态度好不好、愿不愿意学"。遇到不会的问题，诚实说"这个我不太清楚，但我猜是 XX 方向，面试后我去查一下"——比硬编强 100 倍。
2. **项目经历 > 八股**：中小厂大概率花 70% 的时间聊你的项目，30% 的时间问八股。如果一个八股你没背到但项目讲得很清楚，大概率还是过。
3. **反问环节不要浪费**：面试最后面试官会说"你有什么想问我的吗"——不要说"没有"。准备 2 个问题，比如"团队目前主要做什么方向？实习生入职后会负责什么？"——显示你真的对这个岗位感兴趣。
4. **不用穿正装**：中小厂程序员面试穿个干净 T 恤就行。穿西装反而会让面试官觉得你格格不入。

---

> **最后：** 这份文档大概就是你实际会被问到的难度了。如果上面这些你都能用自己的话讲清楚，中小厂日常实习基本稳了。如果有些地方还很虚——别焦虑，你现在大二，大部分人到了大三才勉强能答这些。你现在准备，已经领先了。
